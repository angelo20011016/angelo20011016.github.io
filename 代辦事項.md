# 待辦事項：讓 Hero 介紹等靜態內容可在後台編輯

這是一個用於將網站靜態內容（例如 Hero 區塊的介紹文字、標題等）從硬編碼改為可從後台編輯的計畫。

## 步驟：

### 1. 識別 Hero 區塊的內容來源
- [x] 確定 `frontend/components/sections/HeroSection.tsx` 中哪些內容是需要變為可編輯的。
  - **Hero 背景圖片 URL** (`hero_background_image_url`)
  - **個人照片 URL** (`hero_personal_photo_url`)
  - **主標題** (`hero_main_title`, e.g., "Angelo")
  - **副標題** (`hero_subtitle`, e.g., "Creative Developer")
  - **個人簡介內容** (`hero_bio_content`, 將支援 Markdown)
  - **按鈕 1 標籤** (`hero_button_1_label`, e.g., "查看作品集")
  - **按鈕 2 標籤** (`hero_button_2_label`, e.g., "與我聯繫")

### 2. 設計後端資料模型和 API
- [x] **定義資料模型：** 創建一個 Pydantic 模型 `HeroSettings` (或 `StaticContentConfig` 的一部分)
    - `hero_background_image_url`: `Optional[str]`
    - `hero_personal_photo_url`: `Optional[str]`
    - `hero_main_title`: `str`
    - `hero_subtitle`: `str`
    - `hero_bio_content`: `str` (用於 Markdown 內容)
    - `hero_button_1_label`: `str`
    - `hero_button_2_label`: `str`
- [x] **設計 API 端點：**
    - `GET /api/settings/hero`: 獲取 Hero 區塊的設定。
    - `PUT /api/settings/hero`: 更新 Hero 區塊的設定 (需要管理員權限)。

### 3. 實作後端服務
- [x] **創建 MongoDB 集合：** 在 MongoDB 中建立一個用於儲存網站設定或靜態內容的集合（例如 `settings` 或 `static_content`）。此集合應該只包含單一文件，用於儲存所有靜態設定。
- [x] **創建後端服務文件：** 在 `services/` 目錄下創建一個新的 Python 服務文件（例如 `static_content_service.py`），負責處理與該 MongoDB 集合的互動（讀取和寫入）。
- [x] **創建後端路由文件：** 在 `routes/` 目錄下創建一個新的 Python 路由文件（例如 `static_content.py`），定義用於操作靜態內容的 FastAPI 端點，並使用 `static_content_service.py` 中的服務。

### 4. 實作前端服務
- [x] **創建前端服務文件：** 在 `frontend/services/` 目錄下創建一個新的 TypeScript 服務文件（例如 `staticContentService.ts`），封裝與後端 `/api/settings/hero` 端點的互動（獲取和更新）。

### 5. 創建管理後台介面
- [x] **創建管理表單組件：** 在 `frontend/components/admin/` 目錄下創建一個新的 React 組件（例如 `HeroContentForm.tsx`），它將包含用於編輯 Hero 區塊內容的表單欄位。此表單將使用 `SimpleMdeReact` 編輯器來編輯較長的文本內容（如果需要）。
- [x] **將管理表單整合到 Admin 頁面：** 修改 `frontend/app/admin/page.tsx`，在現有的管理區塊下方添加 `HeroContentForm` 組件。

### 6. 修改 Hero 區塊前端組件
- [x] **更新 `frontend/components/sections/HeroSection.tsx`：** 修改此組件以從 `staticContentService.ts` 獲取 Hero 區塊的內容，而不是使用硬編碼的內容。

### 7. 驗證
- [x] 重新啟動前後端應用程式。
- [x] 登入後台，並嘗試編輯 Hero 區塊的內容。
- [x] 檢查前端顯示的 Hero 區塊是否正確反映了後台的更改。

---
**提示：**

*   所有的 API 請求都應該包含身份驗證 token (如果 Hero 內容的管理是管理員專屬的)。
*   在前端組件中，處理載入狀態和錯誤提示。
*   考慮對敏感或格式要求嚴格的內容進行輸入驗證。
