## ⚙️ 技術實現的核心總結

這個網站的技術挑戰在於**高性能的動態展示**，需要採用分離式架構。

### 1. 🥇 前端視覺與性能 (Frontend Mastery)

這是實現流暢視覺效果的關鍵：

| 技術點 | 具體用途 | 關鍵庫/框架 |
| :--- | :--- | :--- |
| **頂級動畫引擎** | 實現影片中所有**流暢的文字移動、畫面切換**和**變形**，必須保證 60 FPS 運行。 | **GSAP (GreenSock)**：專注於 `Timeline` 和 `ScrollTrigger` 插件。 |
| **伺服器端渲染** | 確保網站載入速度快，**避免長久的白屏時間**，優化 SEO。 | **Next.js**：使用其 SSR/SSG 能力。 |
| **組件化結構** | 結構化管理複雜的 UI 和動態組件，易於維護和擴展。 | **React**：作為 Next.js 的基礎。 |
| **可選 3D 渲染** | 如果要實現開頭那樣的**高擬真 3D 手錶模型**互動。 | **Three.js / WebGL** |

### 2. 🐍 後端數據與邏輯 (Backend Logic)

* **技術選型：** 您熟悉的 **Python (Django 或 Flask)**。
* **職責：** 專門負責提供 **REST API 服務**。例如，管理您的作品集資料、個人簡介內容、處理訪客的聯絡表單提交等。
* **分工模式：** Python 後端專注於數據處理，Next.js 前端透過 API 獲取資料並渲染。

## 🎨 設計原則與視覺呈現

技術只是工具，要讓網站看起來「高級」，設計原則同樣重要。

| 設計重點 | 影片中的體現 | 實現方式 |
| :--- | :--- | :--- |
| **極致排版 (Typography)** | 巨大的、特粗的黑白文字，文字本身就是視覺元素。 | 使用 Web Fonts，並透過 GSAP 精確控制文字的動畫進場、放大與捲動效果。 |
| **極簡風格與對比** | 大量的**黑白灰**和**留白**，僅用少量金色（手錶）點綴。 | 嚴格控制 CSS 顏色和間距，讓使用者視覺焦點集中在核心內容和動態上。 |
| **視覺敘事流程** | 頁面內容以全屏或半屏的區塊，分段、連貫地出現。 | 利用 Next.js 路由或 GSAP `ScrollTrigger` 實現優雅的區塊間轉換。 |
